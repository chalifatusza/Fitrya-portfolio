## Deployment & Database Setup

Follow these steps to deploy and fully connect your portfolio:

### Step 1: Create Supabase Tables
Go to your **Supabase Dashboard** -> **SQL Editor** -> **New Query**, and execute this schema script:

```sql
-- 1. Create admins table with porto_ prefix
CREATE TABLE IF NOT EXISTS porto_admins (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  username TEXT UNIQUE NOT NULL,
  password TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. Create portfolios table with porto_ prefix
CREATE TABLE IF NOT EXISTS porto_portfolios (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  image_url TEXT,
  image_name TEXT, -- Stores fileName for Supabase storage cleanup
  project_url TEXT,
  technologies JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### Step 2: Set Up Supabase Storage Bucket
1. Inside the **Supabase Dashboard**, navigate to **Storage**.
2. Click **New Bucket**.
3. Name it exactly **`my-porto`**.
4. Toggle the **Public** switch **ON** (so portfolio images are publicly viewable).
5. Click **Create**.

### Step 3: Configure Environment Variables
You need to set up the following environment variables (locally in a `.env` file, or in the Vercel Dashboard at **Project Settings -> Environment Variables**):

```env
# JWT Secret for local admin session signing
JWT_SECRET=supersecretjwtkeyforadminauth123

# Supabase Credentials (obtain from Project Settings -> API)
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_KEY=your-supabase-service-role-key
```
> [!IMPORTANT]
> Always use the **`service_role`** secret key as `SUPABASE_KEY` for the serverless function. This ensures the backend has permission to write/delete items in your tables and storage without requiring complicated public Row Level Security (RLS) configurations.

### Step 4: Run Locally
You can run both the frontend and backend servers concurrently on your machine:

1. **Start the Backend API** (Terminal 1):
   ```bash
   npm run start
   ```
   *Express server will run on http://localhost:5000*

2. **Start the Frontend Development Server** (Terminal 2):
   ```bash
   npm run dev-frontend
   ```
   *Vite frontend will run on http://localhost:5173, proxying API requests automatically to port 5000.*
